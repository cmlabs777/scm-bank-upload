# SCM 가계부 업로드 시스템 작업 순서

## 0. 작업 원칙

- 모든 산출물은 `scm` 폴더 안에서만 만든다.
- `voc-radar`의 기존 파일, `gas` 폴더, `idea mock` 폴더 등 다른 작업 영역은 수정하지 않는다.
- GitHub에 올릴 때도 `voc-radar` 프로젝트와 섞지 않고, SCM 전용 비공개 저장소 또는 독립 폴더로 관리한다.
- 커밋 대상은 `scm` 프로젝트 파일만 포함한다.
- 민감 정보가 들어갈 수 있는 엑셀 원본, 계좌번호, Google Apps Script 배포 URL, API 키는 공개 커밋하지 않는다.
- 페이지는 GitHub를 통해 배포된 URL로만 연다.
- 원본 엑셀 파일은 브라우저에서 파싱한 뒤 저장하지 않는다. 최종 데이터는 Google Sheet에만 적재한다.
- 현재 루트 저장소 remote는 `voc-radar`에 연결되어 있으므로, SCM은 별도 private repository 또는 `scm` 내부 독립 git 저장소로 분리한다.
- 이 PC에는 GitHub CLI인 `gh`가 설치되어 있지 않으므로, GitHub 권한 확인과 repo 생성은 `git` 인증 또는 브라우저/GitHub 웹에서 확인한다.

## 1. 목표

카카오뱅크 거래내역 엑셀 파일을 업로드하면 출금 거래만 추출하고, 사용자가 유형을 확인/수정한 뒤 Google Sheet의 `출금내역` 탭 raw data 영역 하단에 자동 적재하는 HTML 기반 도구를 만든다.

별도 유형 관리 화면을 제공하고, 유형 목록은 Google Sheet의 `gas 유형` 탭에서 읽고 저장한다.

## 2. 대상 Google Sheet 구조

- 문서 URL: `https://docs.google.com/spreadsheets/d/1XVJWlIeyo3zvugaJxMJcRxZPkqfxNadDROKwB3xDd-4/edit?gid=0#gid=0`
- 사용 탭:
  - `출금내역`: 최종 raw data 적재 대상
  - `gas 유형`: 유형 목록 저장 대상
- `gas 유형` 탭 구조:
  - `A1`: `유형`
  - `A2`부터: 유형명 목록
  - `B열`: 여백 또는 비고용으로 비워둔다.
  - `C1:E1`: 자동분류 규칙 영역 헤더
  - `C2:E`: 자동분류 키워드, 추천 유형, 설명
- `출금내역` raw data 기본 컬럼:
  - `입출금구분`
  - `월구분`
  - `일자`
  - `금액`
  - `내용`
  - `비고`

## 3. 엑셀 파싱 규칙

- 카카오뱅크 엑셀은 11행이 테이블 헤더이고 12행부터 거래 데이터다.
- 필요한 컬럼:
  - `거래일시`
  - `구분`
  - `거래금액`
  - `거래 후 잔액`
  - `거래구분`
  - `내용`
  - `메모`
- 작업 대상:
  - `거래금액`이 마이너스인 행만 포함한다.
  - 입금 또는 양수 금액 행은 제외한다.
- Google Sheet 적재 시:
  - `입출금구분`: `출금`
  - `월구분`: `일자` 셀을 기준으로 `=YEAR(D행)&"-"&TEXT(MONTH(D행),"00")` 수식을 넣는다.
  - `일자`: 거래일시 기준 실제 Date 값. 표시 형식은 `yyyy. m. d`
  - `금액`: 마이너스 부호를 제거한 양수 금액
  - `내용`: 사용자가 선택한 유형. 즉, 이 프로젝트에서 말하는 `유형`은 `출금내역` 탭 raw data의 `내용` 컬럼 값이다.
  - `비고`: 엑셀의 `내용`과 `메모`를 기준으로 구성

## 4. 중복 업로드 방지 규칙

이미 업로드한 엑셀을 다시 선택할 수 있으므로, 업로드 전 기존 `출금내역` raw data와 비교해서 중복 거래를 제외한다.

추천 룰은 날짜 기준을 기본으로 하되, 같은 날짜에 새 거래가 추가되는 상황을 막지 않기 위해 거래키를 함께 사용한다.

### 4.1 기본 판정

- Google Sheet의 기존 `출금내역` raw data에서 가장 최신 `일자`를 읽는다.
- 엑셀에서 읽은 출금 거래 중 기존 최신 일자보다 이전 날짜의 거래는 기본 제외한다.
- 기존 최신 일자보다 이후 날짜의 거래는 신규 거래로 표시한다.
- 기존 최신 일자와 같은 날짜의 거래는 바로 제외하지 않고 거래키로 한 번 더 비교한다.

### 4.2 같은 날짜 거래키 비교

같은 날짜 거래는 아래 값을 합쳐 만든 `거래키`가 기존 raw data에 있으면 제외하고, 없으면 신규 거래로 표시한다.

- `일자`
- `금액`
- 엑셀의 `내용`
- 엑셀의 `메모`

거래키 예시:

```text
2026-05-31|15900|하이파킹|성동공업고 공영주차장
```

### 4.3 시트 저장 방식

- `출금내역` raw data에 보조 컬럼 `업로드키`를 추가해서 저장하는 방식을 추천한다.
- 이미 H열 이후가 보조 메모로 쓰일 수 있으므로, 실제 구현 시 raw data 헤더 오른쪽의 빈 컬럼을 찾아 `업로드키` 컬럼을 만든다.
- 사용자가 보는 기본 컬럼은 그대로 유지하고, `업로드키`는 중복 방지용 내부 값으로 사용한다.
- 기존 데이터에 `업로드키`가 없는 경우에는 `일자`, `금액`, `비고`를 기준으로 가능한 범위에서 임시 키를 만들어 비교한다.

### 4.4 업로드 화면 표시

- 중복으로 판정된 행은 기본적으로 업로드 대상에서 제외한다.
- 화면에는 `중복 제외` 상태로 접어두거나 별도 필터에서 볼 수 있게 한다.
- 사용자가 필요하면 특정 행을 수동으로 다시 포함할 수 있게 한다.
- 최종 업로드 버튼에는 `신규 N건 업로드, 중복 M건 제외`를 표시한다.

## 5. 페이지 구성

### 5.1 첫 화면

- `업로드하기` 버튼을 제공한다.
- `유형` 버튼을 제공한다.
- 사용자는 첫 화면에서 거래내역 업로드 또는 유형 관리로 이동할 수 있다.

### 5.2 업로드 화면

- 엑셀 파일 선택 입력을 제공한다.
- 엑셀을 선택하면 출금 거래만 행 목록으로 표시한다.
- 이미 업로드된 것으로 판정된 거래는 기본 업로드 대상에서 제외한다.
- 각 행의 우측에 유형 드롭다운을 표시한다.
- 드롭다운 옵션은 `gas 유형` 탭에서 불러온다.
- 자동 분류가 가능한 경우 기본 유형을 미리 선택한다.
- 자동 분류가 애매한 거래는 사용자가 직접 드롭다운에서 선택한다.
- `업로드` 버튼을 누르면 검수된 행을 Google Sheet `출금내역` 탭 raw data 하단에 전송한다.

### 5.3 유형 관리 화면

- 현재 유형 목록을 `gas 유형` 탭에서 불러와 표시한다.
- 유형 추가, 수정, 삭제가 가능해야 한다.
- 저장 버튼을 누르면 `gas 유형` 탭을 아래 구조로 갱신한다.
  - `A1`: `유형`
  - `A2`부터: 저장된 유형 목록

## 6. 구현 방식

### 6.1 프론트엔드

- `scm` 폴더 안에 독립 HTML 프로젝트로 구성한다.
- 우선 단일 HTML 또는 가벼운 정적 파일 구조로 시작한다.
- 엑셀 파싱은 브라우저에서 처리한다.
- 후보 라이브러리:
  - `SheetJS/xlsx`: 브라우저에서 `.xlsx` 파일 읽기

### 6.2 Google Sheet 연동

- Google Apps Script Web App을 API처럼 사용한다.
- HTML 페이지에서 Apps Script Web App URL로 요청을 보낸다.
- GitHub Pages에서 열린 HTML은 Apps Script와 origin이 다르므로 CORS 제약을 고려한다.
- 조회성 요청은 JSONP 또는 CORS 제약이 없는 GET 패턴을 우선 사용한다.
- 저장성 요청은 hidden iframe form POST 또는 simple POST 후 재조회 검증 방식으로 구현한다.
- 필요한 API 기능:
  - 유형 목록 조회
  - 유형 목록 저장
  - 기존 출금 거래키 조회
  - 출금 raw data append
- 자동분류 규칙 조회
- 자동분류 규칙 저장
- Apps Script는 대상 Google Sheet에 붙여서 관리한다.

### 6.3 자동 분류

- 자동분류 규칙은 `gas 유형` 탭 우측 영역에 저장한다.
- A열은 유형 목록 전용이므로 자동분류 규칙 저장 시 건드리지 않는다.
- 추천 저장 위치:
  - `C1`: `키워드`
  - `D1`: `추천유형`
  - `E1`: `설명`
- 예시:
  - `메가`, `커피`, `배민`, `쿠팡이츠` -> `외식비`
  - `쿠팡`, `마트`, `편의점` -> `생활비`
  - `인터넷`, `TV` -> `인터넷TV`
  - `보험`, `주차`, `쏘카` -> `차량비`
- 화면에서는 유형 목록 영역과 자동분류 규칙 영역을 분리해서 보여준다.
- 최종 업로드 전 사용자가 항상 드롭다운으로 수정할 수 있어야 한다.

### 6.4 작업계획 점검 결과

- GitHub URL 배포 조건과 Google Apps Script 쓰기 요청은 CORS 설계를 미리 반영해야 한다.
- 원본 엑셀 파일은 저장소에 포함하지 않는다. `.gitignore`에서 `.xlsx`, `.xls`, `.csv`를 제외한다.
- private repository와 GitHub Pages 조합은 계정/플랜에 따라 제한될 수 있으므로, 배포 전 GitHub Pages 사용 가능 여부를 확인한다.
- Google Sheet 원본 URL이 외부에 노출되어도 직접 접근할 수 없도록 Google Sheet 일반 액세스는 `제한됨`으로 둔다.
- `gas 유형` 탭의 A열은 유형 목록 전용으로 유지하고, 자동분류 규칙은 C열 이후에만 저장한다.
- Google Sheet 일반 액세스는 `제한됨`으로 둔다.
- Apps Script Web App은 `Execute as: Me`로 배포해서 시트 소유자 계정 권한으로 제한된 Google Sheet를 읽고 쓴다.
- 여러 컴퓨터에서 로그인 없이 HTML URL로 사용할 수 있도록 Web App 호출 자체에는 별도 토큰 인증을 두지 않는다.

## 7. 개발 순서

1. `scm` 폴더 안에 프로젝트 기본 파일을 만든다.
   - `index.html`
   - `styles.css`
   - `app.js`
   - `apps-script/Code.gs`
   - `.gitignore`
   - `README.md`

2. 엑셀 파싱 기능을 만든다.
   - 11행 헤더 인식
   - 12행부터 데이터 읽기
   - `거래금액 < 0` 필터링
   - 날짜, 월구분, 금액 정규화

3. 업로드 화면 UI를 만든다.
   - 파일 선택
   - 거래 목록 테이블
   - 유형 드롭다운
   - 업로드 버튼
   - 오류/성공 상태 표시

4. 유형 관리 화면 UI를 만든다.
   - 유형 목록 조회
   - 추가/수정/삭제
   - 자동분류 규칙 조회
   - 자동분류 규칙 추가/수정/삭제
   - 저장 버튼

5. Google Apps Script API를 만든다.
   - `getTypes()`
   - `saveTypes(types)`
   - `getRules()`
   - `saveRules(rules)`
   - `getExistingWithdrawalKeys()`
   - `appendWithdrawals(rows)`
   - `gas 유형` 탭이 없으면 자동 생성
   - `업로드키` 컬럼이 없으면 자동 생성
   - `출금내역` 탭 raw data 하단에 append

6. HTML과 Apps Script를 연결한다.
   - Web App URL 설정 위치 만들기
   - 유형 불러오기
   - 유형 저장
   - 자동분류 규칙 불러오기
   - 자동분류 규칙 저장
   - 기존 출금 거래키 불러오기
   - 출금내역 업로드

7. 실제 카카오뱅크 엑셀로 로컬 테스트한다.
   - `scm` 폴더 안의 엑셀 파일 기준으로 파싱 확인
   - 출금만 표시되는지 확인
   - 금액 부호 제거 확인
   - 월구분/일자 변환 확인
   - 재업로드 시 중복 거래가 제외되는지 확인

8. Google Sheet 테스트 적재를 진행한다.
   - 테스트 행 1~2개로 append 확인
   - raw data 영역 하단 위치 확인
   - 같은 파일 재업로드 시 신규 0건, 중복 제외 N건으로 표시되는지 확인
   - 집계 수식이 깨지지 않는지 확인

9. GitHub 비공개 저장소 준비
   - SCM 전용 private repository 생성
   - `voc-radar`와 구분되는 저장소명 사용
   - 예시: `scm-bank-upload`
   - 엑셀 원본과 민감 설정은 `.gitignore` 처리

10. 커밋 및 업로드
    - `scm` 프로젝트 파일만 커밋
    - 민감 정보 제외 확인
    - private repository로 push

## 8. 확인할 것

- GitHub 저장소 이름
- Apps Script 배포 방식
- private repository에서 GitHub Pages 배포가 가능한지 여부
- Apps Script Web App 쓰기 요청 보호 방식
- `업로드키` 보조 컬럼을 `출금내역` 탭에 추가해도 되는지 여부
