## 2. 입금내역 탭

### 2.1 구조

`입금내역`은 `출금내역`과 거의 같은 raw data 구조를 가진다.

확인된 raw header 위치:

```text
입출금구분 / 월구분 / 일자 / 금액 / 내용 / 비고
```

시트상 실제 컬럼 위치도 `출금내역`과 동일한 패턴이다.

```text
B: 입출금구분
C: 월구분
D: 일자
E: 금액
F: 내용
G: 비고
```

기존 raw 예시:

```text
입금 / 2025-03 / 2025. 3. 5 / 3,327,000 / 월급-다나 / 2월분
입금 / 2025-03 / 2025. 3. 5 / 3,522,677 / 월급-창모 / 2월분
입금 / 2025-03 / 2025. 3. 29 / 126 / 이자 / 2월분
```

### 2.2 추천 방향

입금내역은 현재 출금 업로드 구조를 거의 그대로 재사용하는 것이 좋다.

다만 출금과 다른 점은 다음과 같다.

- 카카오뱅크 엑셀에서 `거래금액 > 0` 인 행만 대상
- `입출금구분`은 `입금`
- `내용` 컬럼에는 입금 유형을 넣는다.
- `비고`에는 엑셀의 `내용`, `메모`, 필요 시 원천 거래구분을 조합한다.
- 중복키는 출금과 동일하게 `거래일시|금액|내용|메모` 구조를 사용한다.
- `월구분`은 직접 문자열을 쓰지 말고 `일자` 컬럼을 기준으로 수식을 넣는다.

월구분 수식:

```text
=YEAR(D행)&"-"&TEXT(MONTH(D행),"00")
```

### 2.3 입금 유형 관리

현재 `gas 유형`은 출금 유형에 맞춰져 있다.

입금 유형은 출금 유형과 성격이 다르므로 별도 탭을 추천한다.

추천 탭:

```text
gas 입금유형
```

추천 구조:

```text
A1: 유형
A2~: 유형 목록
B열: 비움
C1: 키워드
D1: 추천유형
E1: 설명
```

초기 입금 유형 후보:

- `월급-다나`
- `월급-창모`
- `이자`
- `성과급`
- `복지비`
- `부수입`
- `환급`
- `보험금`
- `교통사고`
- `기타입금`

### 2.4 앱 UI 확장

첫 화면의 `업로드하기`는 거래 방향을 선택하게 바꾸는 것이 좋다.

추천 UI:

```text
업로드하기
  - 출금내역 업로드
  - 입금내역 업로드

유형
  - 출금 유형
  - 입금 유형
```

입금 업로드 화면은 출금 화면과 같은 테이블을 사용하되, 대상 필터만 바꾼다.

```text
출금: 거래금액 < 0
입금: 거래금액 > 0
```

## 3. 투자내역 탭

### 3.1 구조

`투자내역`은 `출금내역`/`입금내역`처럼 단순 raw data 테이블 하나가 아니다.

확인된 영역:

- 적금/상품 영역
- 보유중인 종목
- 매도완료 거래내역
- 보유 집계

대략적인 구조:

```text
적금:
  종류 / 상품 / 시작일 / 종료일 / 월납입액 / 회차 / 투입금액

보유중인 종목:
  종류 / 상품 / 매수일 / 단가 / 수량 / 매수금액 / 평가 또는 손익

매도완료 거래내역:
  종류 / 상품 / 거래일 / 단가 / 수량 / 금액 / 실현손익 등

보유 집계:
  종류 / 상품 / 평균단가 / 수량 / 평가금액 / 손익 등
```

정확한 컬럼 의미는 시트의 수식/사용 습관까지 같이 봐야 한다.

### 3.2 은행 엑셀만으로는 자동화가 어려운 이유

카카오뱅크 거래내역 엑셀은 계좌 입출금만 알 수 있다.

예를 들어 `*투자`로 2,000,000원이 출금된 것은 알 수 있지만, 아래 정보는 알 수 없다.

- 어떤 종목을 샀는지
- 매수 단가
- 수량
- 수수료/세금
- 매도 여부
- 실현손익
- 현재 평가금액

따라서 투자내역은 은행 엑셀 업로드만으로 자동 적재하면 운영 목적에 어긋날 가능성이 높다.

은행 거래내역에서 자동으로 처리할 수 있는 것은 `투자 계좌로 돈을 보냈다` 수준이다. 이 값은 이미 `출금내역`의 `*투자` 유형으로 잡는 것이 맞다.

### 3.3 추천 방향

투자내역은 별도 입력 흐름을 만드는 것이 좋다.

추천 우선순위:

1. **수기 입력형 투자 거래 관리**
   - 종목명, 종류, 거래일, 매수/매도, 단가, 수량을 직접 입력
   - 앱이 금액, 보유수량, 평균단가, 실현손익을 계산

2. **증권사 거래내역 CSV/XLSX 업로드**
   - 나중에 사용 증권사가 정해지면 해당 양식에 맞춰 파서 추가
   - 은행 거래내역과 분리

3. **출금 `*투자`와 투자내역 연결**
   - 카카오뱅크 출금 중 `*투자`는 투자 현금 이동으로 본다.
   - 실제 종목 매수/매도는 투자내역에서 별도 입력한다.
   - 둘을 억지로 하나의 자동분류로 합치지 않는다.

### 3.4 추천 투자 탭 구조 개선

현재 `투자내역` 탭은 한 화면에 여러 표가 배치되어 있어 자동 append가 까다롭다.

자동화를 하려면 raw 탭을 하나 추가하는 것을 추천한다.

추천 새 탭:

```text
투자raw
```

추천 컬럼:

```text
거래구분
종류
상품
거래일
단가
수량
금액
수수료
세금
실현손익
비고
업로드키
```

거래구분 예시:

- `매수`
- `매도`
- `적금납입`
- `적금만기`
- `배당`
- `이자`
- `입출금`

이후 `투자내역` 탭은 `투자raw`를 기준으로 집계/피봇/수식으로 보여주는 대시보드 역할을 하게 하는 것이 좋다.

## 4. 구현 권장 순서

### 1단계: 입금내역 자동화

현재 출금 업로드 기능을 확장한다.

작업 항목:

- `parser.js`에 거래 방향 옵션 추가
  - `withdrawal`: `거래금액 < 0`
  - `income`: `거래금액 > 0`
- `app.js`에 업로드 모드 추가
- GAS에 `appendTransactions` 또는 `appendIncomes` 추가
- `입금내역` 탭의 raw header 탐색 추가
- `gas 입금유형` 탭 생성/조회/저장 추가
- 중복키는 출금과 동일하게 사용
- 월구분 수식/일자 Date 처리 방식은 출금과 동일하게 적용

### 2단계: 입금 유형/자동분류

입금 유형은 별도 관리한다.

작업 항목:

- 유형 관리 화면에 탭 또는 세그먼트 추가
  - `출금 유형`
  - `입금 유형`
- `gas 유형`은 출금 유형으로 유지
- `gas 입금유형`을 새로 사용
- 입금 자동분류 규칙 예시 추가

예시 규칙:

```text
급여, 월급, 다나 -> 월급-다나
급여, 월급, 창모 -> 월급-창모
이자, 캐시백 -> 이자
당근 -> 부수입
환급, 취소, 반환 -> 환급
성과급 -> 성과급
```

### 3단계: 투자 입력 기능

투자는 은행 거래내역 자동화가 아니라 별도 화면으로 만든다.

작업 항목:

- `투자 입력` 화면 추가
- `투자raw` 탭 생성
- 매수/매도/적금납입 입력 폼
- 금액 자동 계산
- `투자내역` 탭은 기존 대시보드로 유지하거나, `투자raw` 기반으로 재구성

### 4단계: 증권사 업로드 지원

필요할 때만 진행한다.

작업 항목:

- 사용 증권사 거래내역 양식 확인
- 증권사별 parser 추가
- 매수/매도/배당/수수료/세금 매핑

## 5. 최종 추천

운영 목적을 기준으로 보면, 다음 방향이 가장 안정적이다.

```text
출금내역: 카카오뱅크 엑셀 출금 자동 적재
입금내역: 카카오뱅크 엑셀 입금 자동 적재
투자내역: 은행 엑셀 자동화 대상이 아니라 별도 투자 거래 입력/증권사 업로드 대상
```

즉, 입금은 출금 기능을 복제/확장하고, 투자는 무리하게 카카오뱅크 엑셀에서 끌어오지 않는 것이 좋다.

## 6. 주의사항

- 입금/출금 모두 `월구분`은 문자열 직접 입력이 아니라 `일자` 기반 수식으로 유지해야 한다.
- `일자`는 실제 Date 값이어야 피봇이 같은 월로 묶인다.
- 유형은 각 raw data의 `내용` 컬럼 값이다.
- 출금 유형과 입금 유형은 분리하는 것이 유지보수에 유리하다.
- 투자 계좌로 이체한 금액과 실제 투자 종목 매수는 다른 사건으로 다뤄야 한다.
